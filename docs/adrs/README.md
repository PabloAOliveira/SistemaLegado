# ADRs - SistemaLegado

Este diretorio centraliza as principais decisoes tecnicas do projeto.

## Lista
- `ADR-001`: Padronizar acesso a dados com helpers e SQL parametrizado.
- `ADR-002`: Alinhar rotas com semantica HTTP/REST.
- `ADR-003`: Configuracao por ambiente e segredo fora do codigo.
- `ADR-004`: Evoluir schema com Flask-Migrate em vez de recriar banco.
- `ADR-005`: Testes E2E com banco SQLite isolado por fixture.
- `ADR-006`: Separar dependencias por ambiente e formalizar quality gate.
